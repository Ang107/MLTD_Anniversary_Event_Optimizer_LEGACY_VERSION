MLTD 9th 周年イベント オプティマイザー

アイドルマスターミリオンライブ！シアターデイズ周年イベント（9th）の効率的なプレイ計画を算出します。

オプティマイザー 使い方 仕様詳細

このツールについて

想定しているプレイヤー

イベントポイント 100万〜700万 を目標に走る方を主な対象としています。

設計の方針

「計画通りにプレイしようとしたのに時間が足りず、想定より少ないポイント・トリガーしか稼げなかった」という事態が起きにくいよう、ライブスキップやリフレッシュ中のチケット収集といった時間短縮の上振れ要素はあえてシミュレーションに含めず、余裕を持った計画を出力することを意識しています。

リフレッシュ中のミッション達成やチケット収集、ブーストのタイミング・種類の使い分けなど、さらに効率を高める余地は残りますが、このツールではなるべく考えることを少なくしつつ、そこそこ効率的に走れることを重視しています。

主な機能

計画の最適化

「▶ 最適化」ボタンを押すと、モードに応じた結果が表示されます。

各種条件をカスタマイズすることで、それに合わせた最適化結果を得ることができます。

最適化モード

  • ポイント最大化：指定された稼働時間の中で獲得できるポイントを最大化するプレイ計画を出力します。
  • 稼働時間最小化：指定された目標ポイントに到達するための稼働時間を最小化するプレイ計画を出力します。

おすすめ楽曲スケジュール

  • 確定：イベント期間中の各日の最適化されたプレイ計画を出力します。
  • 未確定：開始日の最適化されたプレイ計画を出力します。

途中からの利用

「シミュレーション開始日時」「現在の所持ポイント」「現在の所持トリガー」、および開始日に既に済ませた行動（「ログイントリガー（540）」「ミッショントリガー（1000×4）」「ブースト」「周年曲10倍」）を設定することで、イベント途中からでも最適化を実行できます。

設定の保存・読み込み

  • 設定を書き出し：現在の設定をJSONファイルとして保存します。
  • 設定を読み込み：保存したJSONファイルを読み込み、設定を復元します。
  • 設定をリセット：すべての設定を初期値に戻します。

設定はブラウザの localStorage にも自動保存されるため、ページをリロードしても入力内容が保持されます。

結果の共有

最適化を実行すると、結果の右上に「共有」ボタンが表示されます。押すと共有メニューが開き、現在の設定をすべて埋め込んだ共有リンク（URL）を作成できます。

  • リンクをコピー：共有リンクをクリップボードにコピーします。
  • X で共有：最終ポイントと合計稼働時間を添えた投稿文とともに、共有リンクを X（旧Twitter）の投稿画面で開きます。

共有リンクには現在の入力内容がすべて含まれています（サーバーには保存されず、設定はリンク自体に埋め込まれます）。リンクを開くと、その設定が自動で復元されたうえで最適化も自動的に実行され、共有元と同じ結果がそのまま表示されます。設定ファイル（JSON）をやり取りしなくても、URL ひとつで設定と最適化結果を共有・引き継ぎできます。

基本的な使い方

  1. 「オプション」「おすすめ楽曲」「設定」欄の項目を編集する。
  2. 必要に応じて「設定を書き出し」で設定を保存しておく。
  3. 「▶ 最適化」ボタンを押して最適化を実行する。
  4. 最適化結果の各日の行をクリックして展開し、そのプレイ手順に沿ってイベントを走る。
  5. 定期的に初期状態を更新してから最適化を再実行し、計画をアップデートしつつイベントを走る。

シミュレーションはライブスキップを使わない前提で計算しているため、行動詳細にライブスキップは表示されません。実際のプレイでは、ライブスキップを適当なタイミングで使ってかまいません。ライブスキップを使ってもかかる時間が短くなるだけで、計画に大きな影響はありません（むしろ時間に余裕ができます）。

各項目の説明

オプション

項目 説明
最適化モード

ポイント最大化：イベント全体を通じて獲得できる合計ポイントが最大になるようプレイ計画を立てます。

稼働時間最小化：「目標ポイント」に到達するのに必要な1日あたりの稼働時間が最小になるよう計画します。全日の稼働可能時間に均一に乗じる倍率を最小化します。
目標ポイント 稼働時間最小化モードのみ有効です。この値以上のポイントを獲得するためのプレイ計画を立てます。
ブーストモード

通常曲ブースト：ブーストをライブチケット450の通常曲に使用します。スタミナ・ジュエルあたりのポイント効率が良いです。

周年曲ブースト：ブーストを周年曲4倍ライブに使用します。時間あたりのポイント効率が良いです。
おすすめ楽曲スケジュール確定

確定：全13日分のおすすめ楽曲スケジュールが分かっている場合に選択します。下の「おすすめ楽曲」テーブルに全行を入力してください。

未確定：シミュレーション開始日以降のスケジュールが未定の場合に選択します。開始日以前の行のみ入力します。
乱数シード おすすめ楽曲スケジュールが未確定のときのみ有効です。同じシードなら何度実行しても同じ結果になります。
シミュレーション回数 おすすめ楽曲スケジュールが未確定のときのみ有効です。多いほど期待値の精度が上がりますが、計算に時間がかかります。
シミュレーション開始日時 シミュレーションを開始する日と時刻を設定します。開始時刻より前の当日の稼働可能時間は計算対象から除外されます。「現在日時に設定」ボタンで現在の日時を反映できます。
ログイントリガー（540） シミュレーション開始日のログイントリガー（540）を既に受け取った場合に取得済みを選びます。受け取ったトリガーは「現在の所持トリガー」に含まれているものとして扱い、開始日に改めて受け取りません。
ミッショントリガー（1000×4） シミュレーション開始日のおすすめ楽曲ミッション報酬のトリガー（1000×4）を既に受け取った場合に取得済みを選びます。この報酬はおすすめ楽曲を1回ずつプレイすることで得られるため、取得済みとした場合は、その成果（ポイント／トリガー）が「現在の所持ポイント／トリガー」に含まれているものとして扱い、開始日にこのプレイを改めて行いません。
ブースト シミュレーション開始日のブーストを既に使った場合に使用済みを選びます。獲得・消費したポイント／トリガーは「現在の所持ポイント／トリガー」に含まれているものとして扱い、開始日に改めてブーストを行いません。
周年曲10倍 シミュレーション開始日の周年曲10倍ライブを既にプレイした場合にプレイ済みを選びます。獲得したポイントは「現在の所持ポイント」に含まれているものとして扱い、開始日に改めて周年曲10倍を行いません。
現在の所持ポイント シミュレーション開始日時時点で既に持っているイベントポイントです。最終的な合計ポイントに加算されます。
現在の所持トリガー シミュレーション開始日時時点で既に持っているトリガーです。それを考慮して計画が立てられます。

各種時間・スタミナ

項目 説明
前半戦 1800枚収集時間（秒） 前半戦（1〜6日目）にお仕事でチケット1800枚を集めるのにかかる時間です。
後半戦 1800枚収集時間（秒） 後半戦（7〜13日目）にお仕事でチケット1800枚を集めるのにかかる時間です。
周年曲の曲時間（秒） 周年曲の演奏時間（秒）です。ライブ1回あたりの所要時間の計算に使われます。
メニュー遷移（秒） メニューを遷移する（例: ライブ → お仕事）時間です。
楽曲選択画面→曲開始（秒） ライブページの楽曲選択の画面から曲の開始までの遷移時間です。
曲終了→楽曲選択画面（秒） 曲が終わってからライブページの曲の選択画面に戻るまでの時間です。
曲終了→次曲開始（再演）（秒） 再演機能を用いた場合の、曲終了から次の曲開始までの時間です。
スパークドリンク 10/20/30/MAX 個数 所持しているスパークドリンクの個数。必要ジュエル計算時に消費スタミナから差し引かれます。
スタミナ最大量 プレイヤーのスタミナ上限。ジュエルによる回復1回あたりの回復量として使われます。

日ごとの稼働可能時間・リフレッシュ開始時刻

項目 説明
稼働可能時間（時間） その日にゲームをプレイできる時間の上限です。
リフレッシュ開始時刻 リフレッシュタイムが始まる時刻（0〜23時）。8時間のリフレッシュ中はゲームプレイができないとして扱われます。

アイドル別 おすすめ楽曲の曲名・長さ

各アイドルのおすすめ楽曲の曲名と時間（秒）です。曲名は省略可能で、結果の行動詳細に表示されます。時間（秒）はライブにかかる時間の計算に使われます。

おすすめ楽曲プリセット

おすすめ楽曲テーブルの上にあるドロップダウンで、テーブル全体のアイドルの並びを既知の順番に一括で差し替えられます。1枠ずつ入力する手間を省くためのものです。選んだあとに個別のマスを手で修正することもできます。

  • あいうえお順：アイドルを五十音順に並べます。
  • ミリシタキービジュアル登場順（年齢降順）：キービジュアルの登場順（年齢の降順）に並べます。
  • ソロ2周目実装順：ソロ楽曲2周目の実装順に並べます。
  • ランダム：アイドルをランダムに並べます。選ぶと「↺ 再シャッフル」ボタンが表示され、押すたびに並びを引き直せます。

おすすめ楽曲テーブル

各日のおすすめ楽曲に対応するアイドルを選択するテーブルです。

  • 確定モードの場合： 全13日 × 4枠 = 52枠すべてに、重複なくアイドルを割り当ててください。
  • 未確定モードの場合： シミュレーション開始日（設定値）以前の行のみ入力してください。

重複や未入力の枠は赤く表示されます。最適化を実行する前に解消してください。

前提条件

このオプティマイザーは以下の前提のもとで最適化を行います。これらの前提から外れる場合、結果が実際の状況と乖離する可能性があります。

イベント仕様

  • イベント期間は 2026年6月30日（火）〜7月12日（日）の13日間。
  • 前半戦：1〜6日目（〜7月5日）、後半戦：7〜13日目（7月6日〜）。
  • 後半戦から 3倍お仕事 および 周年曲10倍ライブ×2回 が解放される。
  • リフレッシュタイムは 8時間。開始時刻は日ごとに設定可能。

その他の前提条件

  • DX+ 課金済みであること（チケット上限が1800枚を前提）。
  • ライブスキップは使用しない。
  • 各日に ブーストを1回 行い、通常曲 or 周年曲を最低でも10回以上プレイする。（稼働可能時間を超過する可能性がある。）ただし、シミュレーション開始日についてはブースト・周年曲10倍を既に済ませたかどうかを設定で指定でき、済ませたものはその成果が「現在の所持ポイント／トリガー」に含まれているものとして扱い、開始日には改めて行わない。
  • リフレッシュ中にチケットを集める行動は行わない。
  • 「お仕事でチケットを1800枚集める → その日の最速のおすすめ楽曲をチケ450枚ライブで4回プレイ」の行動を一塊のルーティンとして扱う。
  • 「お仕事でチケットを1800枚集める → その日のおすすめ楽曲をチケ450枚ライブで1回ずつプレイ」の行動を1日に1回行う。
  • レベルアップによるスタミナ回復・スタミナ最大量の増加は考慮しない。
  • シミュレーション開始日にログイン・ミッションで貰えるトリガーは、設定の「ログイントリガー」「ミッショントリガー」で取得状況を指定する。未取得とした分はこれから受け取れるものとして計画に含め、取得済みとした分はその成果が「現在の所持ポイント／トリガー」に含まれているものとして扱い、開始日には改めて受け取らない。
  • シミュレーション開始日より前のミッションで貰えるトリガーは受け取り済みとする。

注意事項

  • 最適化アルゴリズムについて
    シミュレーション上で良い近似解が得られますが、最適解を保証するものではありません。
  • 計算結果の正確性について
    最適化ロジックに誤りが含まれる可能性があります。結果に不自然な点がある場合は設定値を確認し、問題が解消しない場合は作者までご連絡ください。
  • 入力値の精度について
    計算結果の品質は入力値の精度に依存します。特に操作時間（ライブ画面遷移・お仕事時間など）は実際に計測した値を入力することをおすすめします。設定値と実際のプレイスピードに差があると、結果と実際の稼働時間がずれることがあります。
  • ゲーム仕様の変更について
    本ツールは公開時点の仕様をもとに作成されています。仕様変更があった場合、ツールへの反映が遅れることがあります。
